# 開始使用 — Mac 版 Speaker Detect

呢份指引係寫俾**完全冇用過 Terminal、冇裝 git 都唔緊要**嘅朋友嘅。
跟住做，大約 5–10 分鐘就可以喺你部 Mac 上面行到個 app。

---

## 你會得到乜

一個黑底全螢幕嘅程式：

```
┌────────────────────────────────────┐
│ ▬▬▬▬▬▬▬▬▬▬▬▬                       │   ← 即時音量 bar
│                                    │
│                                    │
│               B                    │   ← 超大字，顯示而家邊位講緊嘢
│                                    │     (A / B / C...)
│           SPEAKER B                │
│     68% match · analysis #12       │   ← similarity score + debug
│                                    │
│       [A] [B] [C]                  │   ← 所有講者 chips
├────────────────────────────────────┤
│    STOP                   RESET    │
└────────────────────────────────────┘
```

**當講者由 A 變 B 嗰陣：**
- 個 screen 會閃兩下白光
- Mac 右上角會彈 native notification：「Speaker Changed: A → B」
- 仲會 "叮" 一聲（Glass 音效）

## 準備工夫（1 分鐘）

喺 Spotlight 開 **Terminal**（`⌘ + Space`，打 `Terminal`，按 Enter），
然後 copy 下面條 command 落 Terminal，按 Enter：

```bash
python3 --version
```

**預期結果：** 會見到 `Python 3.10.x` 或者更高版本（例如 `Python 3.11.6`）。

> 如果見到 `command not found`，去到最後嘅「常見問題」章節睇點樣裝 Python。

---

## Step 1：下載個 project（完全唔使 git）

1. 喺 **Safari**（或者 Chrome）打開呢條 link：

   https://github.com/yinstagram/Claude-speakers-detaction-/archive/refs/heads/claude/fix-mobile-github-voice-EI0h3.zip

2. Safari 會自動 download 一個 zip file 去 `~/Downloads/`，
   通常會自動解壓成一個 folder，名係咁嘅樣：

   ```
   Claude-speakers-detaction--claude-fix-mobile-github-voice-EI0h3
   ```

3. 喺 **Finder** 入面去 `Downloads`，確認你見到呢個 folder。
   *（如果冇解壓，double-click 個 .zip file 就會解壓。）*

---

## Step 2：開 Terminal 去到個 folder

1. `⌘ + Space` → 打 `Terminal` → Enter。

2. 喺 Terminal 打 `cd` 同**一個 space**（唔好按 Enter）：

   ```bash
   cd 
   ```

3. 喺 Finder 揾返 Step 1 嗰個 folder，**用滑鼠拖佢落 Terminal window**。
   Terminal 會自動幫你填咗成個 path，類似：

   ```bash
   cd /Users/你/Downloads/Claude-speakers-detaction--claude-fix-mobile-github-voice-EI0h3
   ```

4. 按 Enter。

5. 再打呢兩條 command，每條按一次 Enter：

   ```bash
   cd mac
   ls
   ```

   **預期結果：** 你會見到呢啲 file：

   ```
   README.md  requirements.txt  run.sh  speaker_detect.py  開始使用.md
   ```

如果見到 `run.sh`，即係你嚟啱地方喇。

---

## Step 3：第一次行（要等 3–5 分鐘）

喺 Terminal 打：

```bash
./run.sh
```

按 Enter。

**第一次會做三件事，會慢少少（3–5 分鐘），呢個係正常嘅：**

1. 整一個 Python 虛擬環境（venv）
2. 用 `pip` 裝 `torch`、`speechbrain`、`PyQt6`、`sounddevice` 等等
3. Download ECAPA-TDNN model（~20 MB）

你會見到類似咁嘅 output，不斷有新 line 出：

```
==> 工作目錄：/Users/你/Downloads/.../mac
==> 建立 Python 虛擬環境...
==> 安裝 dependencies（要等 3-5 分鐘，呢個係正常嘅）...
Collecting torch>=2.2.0
  Downloading torch-2.2.1-....whl (150 MB)
     ━━━━━━━━━━━━━━━━━━━━  45.2/150 MB 3.1 MB/s
...
==> 啟動 Speaker Detect...(第一次要等 ECAPA-TDNN model 下載)
[engine] using device: mps
```

**最後會有個黑底嘅 window 彈出嚟**，中間寫住大大個 `?`，
下面寫 `LOADING MODEL...` → 跟住變 `TAP START`。

> 如果 Gatekeeper 阻咗你執行 `./run.sh`，試下改用：
> ```bash
> bash run.sh
> ```

---

## Step 4：俾咪高風權限

第一次按 START 嗰陣，macOS 會彈一個 dialog 問：
**「Terminal would like to access the microphone.」**

👉 **一定要㩒「OK」**。

如果你唔小心 click 錯咗「Don't Allow」，冇得彈第二次，要手動開返：

1. 開 **System Settings**
2. 去 **Privacy & Security** → **Microphone**
3. 揾到 **Terminal**，將個 toggle 開到**綠色**
4. 返去 Terminal 按 `Ctrl + C` 停咗個 app，再行 `./run.sh`

---

## Step 5：點用

1. 喺個 app window 按 **START**（黑底綠字嗰個掣）。
2. 中間會變做 `...` 然後 `LISTENING`。
3. **你講兩三句嘢**（講乜都得，可以數 1-2-3-4-5-6-7-8）。
4. 你會見到中間變成 **綠色嘅 A**，底下寫 `SPEAKER A`。
5. **搵另一個人講兩三句**（或者你自己扮另一個聲都可以）。
6. 應該會見到：
   - 個 screen **閃兩下白光** ⚡
   - 中間由 `A` 變 **藍綠色嘅 B**
   - Mac 右上角彈通知：**Speaker Changed: A → B**
   - 聽到 "叮" 一聲

底下嘅 debug line 會顯示 similarity 分數，類似：

```
-22dB | A=0.81 B=0.42 | thr=0.55
```

> `A=0.81` = 而家呢段聲同 A 嘅 profile 有 81% 相似；
> `thr=0.55` = 要高過 55% 至當同一個人。

### 其他掣

- **STOP**：停止錄音，但**保留** speaker profiles。再㩒 START 會繼續認返之前嘅人。
- **RESET**：**清空曬**所有 speaker profiles，由頭嚟過。

## 常見問題

### 「command not found: python3」

即係部 Mac 冇裝 Python。打呢條 command 去裝 Xcode Command Line Tools（會連埋 python3、git 都裝埋）：

```bash
xcode-select --install
```

彈 dialog 嗰陣按 **Install**，等佢裝完（10-15 分鐘），再試返 Step 3。

### 「Permission denied」 when running `./run.sh`

打：

```bash
chmod +x run.sh
./run.sh
```

### 「MODEL FAILED」 或者卡住喺 `Loading ECAPA-TDNN...`

通常係網絡問題。試：

1. 按 `Ctrl + C` 停咗 Terminal
2. 確認有 Wi-Fi
3. 再行 `./run.sh`

如果成日唔得，刪除 cache 然後再試：

```bash
rm -rf ~/.cache/speaker_detect
./run.sh
```

### 個 energy bar 完全唔郁 / 講嘢都冇反應

代表 app 收唔到咪高風訊號。檢查：

1. **System Settings → Privacy & Security → Microphone → Terminal** 有冇開
2. **System Settings → Sound → Input** 揀啱咗正確嘅 microphone
3. 對住個 mic 講，睇下 Input level bar 有冇郁

### 兩個人一直被當成同一個

開 `speaker_detect.py`（可以用 TextEdit 或者任何 editor），
揾到最上面呢一行：

```python
SIMILARITY_THRESHOLD = 0.55
```

改做 `0.60` 或者 `0.62`（越高越嚴格），save 之後再行 `./run.sh`。

### 同一個人被拆做 A、B、C、D...

同上，改做 `0.50` 或者 `0.48`（越低越鬆），save 之後再行 `./run.sh`。

### 點樣完全關閉個 app？

- 關 Terminal window；或者
- 喺 Terminal 按 `Ctrl + C`；或者
- 直接 close 個 app window（右上角紅色圓點）
